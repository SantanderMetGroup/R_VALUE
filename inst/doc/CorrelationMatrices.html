1. Introduction

This is a worked example on how to compute correlation matrices from the outputs of the different VALUE experiments and datasets. corrMat.VALUE is a specific function to this aim, as part of the R.VALUE package for data validation. The main features have been discussed previously by the validation paper working group, and have been implemented as arguments of the function. As a result, different correlation methods can be indicated (e.g.pearson for temperatures and spearman for precipitation). A (circular) moving average daily climatology can be automatically calculated for each data series, considering a specific window width (60 days by default, centered around lag 0). This is used for removing the seasonal cycles when applicable. An additional argument allows to specify the maximum number of missing values allowed, being ommited from the analysis those data series above the threshold.

The function corrMat.VALUE is intended to handle the predictions files as directly downloaded from the VALUE validation portal, without the need of further preprocessing.

1.1. Installing R.VALUE

R.VALUE is available on GitHub. The most direct way of installing the package is using the install_github function from package devtools. Note that a special development branch called spvalPaper has been created for testing purposes, aimed at containing different functions developed for the spatial validation paper before these are agreed to be included in the master branch.

require(devtools)
install_github("SantanderMetGroup/R_VALUE")

1.2. Example datasets (observations and predictions)

Tha data from the different experiments already available can be downloaded from the VALUE-COST Validation Portal, under the Validation Tab (requires a valid username and password for access). The option Download prediction must be chosen. All experiments have an associated observational dataset, as indicated in the Experiments Table.

In this example, we will use the predictions from Experiment_1a. In particular, the outputs of the direct analogs method. For the sake of example reproducibility, both the reference station datasets (VALUE_ECA_86_v2) and two predictions zip files (deterministic and stochastic) as directly downloaded from the portal1, have been included in the installation of the R.VALUE package.

Path to the VALUE_ECA_86_v2 dataset (this will vary depending on the installation directory):

# Path to ECA dataset file
obs.file <- file.path(find.package("R.VALUE"), "example_datasets", "VALUE_ECA_86_v2.zip")

Path to the predictions datasets (this will vary depending on the installation directory):

# Display paths to predictions files
list.files(file.path(find.package("R.VALUE"), "example_datasets"),
           pattern = "predictions_portal_exp1a",
           full.names = TRUE)

1.4. Loading predictions

Predictions are loaded with the loadValuePredictions function (see help loadValuePredictions for further details). However, this will be internally done by corrMat.VALUE if we specify a path to the predictions file via the predictions.file argument. Otherwise, the correlation matrix will be undertaken on the observational dataset, as illustrated in the next section. Nevertheless, an example on loading a stochastic predictions file (as downloaded from the VALUE validation portal) is next shown:

2. Correlation matrices

2.1. Correlation matrix of the observations:

The following instruction will compute the correlation matrix considering the minimum temperature observations previosuly loaded (stationObj = obs and predictions.file = NULL arguments), for the entire year (season = "annual"). In addition, only those stations with less than 25% missing data will be considered (max.na.prop = 0.25, in this case all the 86 stations fulfill the condition). The type of correlation will be pearson (method = "pearson"). Furthermore, the seasonal cycle will be removed (deseason = TRUE). To this aim, a daily climatology of reference will be first computed for each station, considering a 60-day wide moving window (window.width = 60).

Note that some of these arguments may be ommited as they correspond to the default values. For further information see help("corrMat.VALUE")

a <- corrMat.VALUE(stationObj = obs,
                   season = "annual",
                   max.na.prop = 0.25,
                   predictions.file = NULL,
                   method = "pearson",
                   deseason = TRUE,
                   window.width = 60)
## [2016-02-15 10:48:22] - Removing seasonal cycle...
## [2016-02-15 10:48:49] - Done.

The path to the predictions file need to be specified in the predictions.file argument. In this example, we use the built-in example file of deterministic predictions, as downloaded from the validation portal:

In this example, we will compute correlations for wintertime (season = DJF), considering a moving window of 30 days (window.width = 30):

2.2.2. Correlation matrix of stochastic predictions

The process is similar, but in this case there is an optional argument in order to indicate whether the different realizations should be aggregated before computing the correlations (also before computing the optional removal of the seasonal cycle), or fater computing the correlations (i.e., aggregation of the correlation coefficients for each realization). Two examples follow:

prdfile.stoch <- list.files(file.path(find.package("R.VALUE"), "example_datasets"),
                            pattern = "predictions_portal_exp1a_stochastic",
                            full.names = TRUE)

All members are aggregated before computing the correlation:

c.before <- corrMat.VALUE(stationObj = obs,
                          predictions.file = prdfile.stoch,
                          season = "DJF",
                          type = "before",
                          max.na.prop = 0.25,
                          method = "pearson",
                          deseason = TRUE,
                          window.width = 30)
## [2016-02-15 11:31:22] - Loading predictions...
## [2016-02-15 11:31:43] - Done.

The same method, but all members are aggregated after computing the correlation:

                          deseason = TRUE,

2.3. Plotting correlations

The following example compares the result from the two types of aggregation (it requires the R package lattice):

c.before[lower.tri(c.before)] <- c.after[lower.tri(c.after)]
diag(c.before) <- NA
# Graphic pars
station.labels <- attr(c.before, "station_names")
scales.list <- list(x = list(labels = station.labels, rot = 90,
                             at = seq(1,ncol(c.before),1), cex = .5),
                    y = list(labels = station.labels,
                             at = seq(1,ncol(c.before),1), cex = .5))
require(lattice)
## Loading required package: lattice
levelplot(c.before, xlab = "r - AFTER", ylab = "r - BEFORE", main = "correlation matrix",
          scales = scales.list, at = seq(0,1,.05))
